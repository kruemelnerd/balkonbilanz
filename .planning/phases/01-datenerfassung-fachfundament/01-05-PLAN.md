<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test harness -> app runtime | Testcode darf nicht an produktivem Pfad vorbeimogeln |
| browser automation -> local db state | E2E interagiert mit realer Persistenz und muss deterministisch bleiben |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | T | Component/E2E tests | mitigate | Verbot von `setContent`-Abkuerzungen; Tests muessen ueber `page.goto`/render echte App-Komponenten nutzen |
| T-01-05-02 | R | BDD step definitions | mitigate | Steps auf reproduzierbare, beobachtbare UI-Ergebnisse mappen und Failure-Assertions explizit machen |
| T-01-05-03 | D | Test runtime dependencies | mitigate | Fehlende Runner-Abhaengigkeiten/Skripte in package.json herstellen, damit Tests nicht aus Umgebungsgruenden ausfallen |
| T-01-05-04 | S | Synthetic fixtures | accept | Testdaten sind lokal, nicht sensitiv; Risiko begrenzt auf falsches Sicherheitsgefuehl, durch obige Mitigations reduziert |
</threat_model>
