Use the `readme-maintainer` skill.

Create or improve the project's `README.md`.

Focus especially on these required parts:

1. Kurzbeschreibung
2. Screenshot oder CLI-Beispiel
3. Installation
4. Quickstart
5. CLI-Commands und Options

For installation, distinguish between production and development setup if they differ.

Use the repository as the source of truth. Do not invent commands, package names, URLs, screenshots, Docker images, options, badges, or licenses.

If information is missing, add precise `TODO:` notes.
